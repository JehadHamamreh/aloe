Creates a new process to be executed in one of the kernel pipelines. The structure pointed by attr has the following options\-:
\begin{DoxyItemize}
\item binary\-\_\-path\-: Path to the shared library where the program code resides
\item pipeline\-\_\-id\-: If the machine has more than one pipeline (e.\-g. in a multi-\/core system, it indicates to which the process will be loaded).
\item exec\-\_\-position\-: Indicates the execution position in the pipeline.
\item finish\-\_\-callback\-: If non-\/null, the kernel will call this function when the process finishes (normally or due to an error) its execution.
\end{DoxyItemize}

\hyperlink{group__process_gaac39f57d3d63eca01423a7ffd95026e1}{rtdal\-\_\-process\-\_\-new()} configures the process parameters and loads it into the pipeline. The process won't execute until a success call to \hyperlink{group__process_ga5c4d6fda8cbcf1f0721e894fde223173}{rtdal\-\_\-process\-\_\-run()} is performed.

The address pointed by arg will be passed to the process as a parameter.

Notes\-: The shared library must conform with the rtdal\-\_\-process\-\_\-t requirements.
